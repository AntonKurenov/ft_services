Sequencing for start the pod:
docker build -t <image_name> .
kubectl apply -f config.yaml
kubectl get pods 			-for information about pods
kubectl logs <POD_NAME>
kubectl delete <POD_NAME>

For deployments info:
kubectl get deploy

1. minikube start
2. minikube delete
3. kubectl get svc

For create pods:
kubectl apply -f config.yaml

For delete pods:
kubectl delete pods <POD_NAME>

For stats about pod:
kubectl logs <POD_NAME>

kubectl get — вывод списка ресурсов
kubectl describe — вывод подробной информации о ресурсе
kubectl logs — вывод логов контейнера в поде
kubectl exec — выполнение команды в контейнере пода
kubectl exec $POD_NAME env - execute the env command in POD
kubectl exec -ti $POD_NAME bash - run the pods terminal 
kubectl get pods - get list of pods
kubectl get services - get list of services

are now by default launch with --vm-driver=virtualbox
it is done by command :
minikube config set vm-driver virtualbox

docker не запустится сразу как только запустился кубер, нужно ввести команду:
eval $(minikube docker-env)

for docker:
docker build -t <name> . (from that directory)

To get inside pod run:
kubectl exec --stdin --tty <POD_NAME> --/bin/sh
if two or more pods are present:
kubectl exec -i -t my-pod --container main-app -- /bin/bash
